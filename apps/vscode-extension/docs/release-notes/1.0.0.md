# KiCad Studio 1.0.0 Release Notes

KiCad Studio 1.0.0 is the first monorepo baseline for the VS Code extension in
`oaslananka/kicad-studio-kit`. It keeps the core KiCad workspace features
independent while synchronizing the optional `kicad-mcp-pro` integration with
the shared repository compatibility matrix.

## MCP Compatibility

The extension negotiates the MCP server during initialize with protocol
`2025-11-25`. It supports `kicad-mcp-pro >=1.0.0 <2.0.0`, recommends the same
range, and records the last observed server card for optimistic UI rendering on
the next launch. Incompatible servers are marked explicitly and do not receive
context pushes, fix queue calls, quality gate calls, or release requests.

## Quality Gates Sidebar

The Quality Gates view surfaces project, placement, transfer, and manufacturing
readiness checks. It renders cached gate results while fresh MCP calls are
running, expands failed or blocked rows into violations, and opens source
locations when the server provides file and line metadata.

## Fix Queue Code Actions

MCP fix queue items with `path` and `line` metadata appear as VS Code Code
Actions near matching diagnostics. Text-only fix queue items continue to render
in the AI Fix Queue tree and can still be applied manually.

## Installer And Profiles

`KiCad: Install kicad-mcp-pro` detects `uvx`, `pipx`, and `pip`, then runs the
selected installer as a visible VS Code Task. The MCP status entry also exposes
a profile picker for the documented `full`, `minimal`, `schematic_only`,
`pcb_only`, `manufacturing`, `high_speed`, `power`, `simulation`, `analysis`,
and `agent_full` profiles.

## Manufacturing Release Wizard

`KiCad: Manufacturing Release Wizard` guides users through variant selection,
project quality pre-flight checks, output confirmation, MCP manufacturing
export, and post-run handling. Blocking gates surface MCP hints when present.

## Configuration And Debugging

`.vscode/mcp.json` has KiCad-aware JSON schema validation. The MCP log viewer
keeps a redacted ring buffer of recent request/response traffic and can save
that log for bug reports without exposing authorization headers or full
home-directory paths.

## CI Coverage

Real-server integration test scaffolding launches
`kicad-mcp-pro==1.0.0` through `uvx` when policy allows it. Mocked unit and
integration tests cover version negotiation, tree mapping, Code Actions,
installer selection, structured errors, protocol metadata, and log redaction.
